Como tratamos dados no Moodle Consultas

Este documento descreve, de forma objetiva, como a aplicação trata dados de cadastro, autenticação e consulta ao Moodle para apoiar o uso da API e das GPT Actions associadas ao projeto.

Última atualização
14 de maio de 2026
Escopo
Página de cadastro, Edge Functions, proxy Moodle e OpenAPI.
Status
Modelo operacional a ser validado pelo responsável jurídico.

1. Quem somos

O Moodle Consultas é uma API de consulta, em modo somente leitura, que permite a usuários autorizados consultar informações acadêmicas do Moodle por meio de GPT Actions. O projeto utiliza Supabase Edge Functions como proxy entre o ChatGPT Action e os Web Services do Moodle.

2. Abrangência desta política

Esta política explica como o projeto trata dados pessoais na página de cadastro, nas Edge Functions register e moodle-proxy, e no schema OpenAPI usado para configurar as GPT Actions.

3. Dados pessoais que podemos tratar

  • Nome, e-mail e chave de API gerada para autenticação.
  • Data e hora de cadastro, status da chave e metadados técnicos de requisição.
  • Dados acadêmicos retornados pelo Moodle conforme as permissões do token técnico configurado pela instituição.

4. Finalidades do tratamento

  • Cadastrar usuários autorizados e gerar chaves de API.
  • Autenticar requisições e permitir consultas acadêmicas autorizadas.
  • Proteger o ambiente contra uso indevido, acessos não autorizados e exposição de segredos.

5. Armazenamento, segurança e direitos

Os dados de cadastro são armazenados no Supabase enquanto forem necessários para autenticação, auditoria operacional e segurança do serviço. O projeto adota controles como Bearer token, validação de chaves ativas, armazenamento de segredos apenas no backend e limitação do proxy a operações de leitura.

Contato e documento completo

A versão completa desta política está mantida no repositório e deve ser revisada pelo responsável jurídico ou encarregado de dados antes da publicação oficial. Caso precise, o link de privacidade também está disponível na página inicial.
