Como tratamos dados no Moodle Consultas

Este documento descreve como o Moodle Consultas coleta, usa, armazena, compartilha e protege dados pessoais ao oferecer acesso à API de consulta ao Moodle por meio de GPT Actions e do painel da plataforma.

Escopo

1. Quem somos

O Moodle Consultas é uma plataforma de API que permite a usuários autorizados consultar, em modo somente leitura, informações acadêmicas do Moodle por meio de GPT Actions personalizadas. A plataforma utiliza Supabase Edge Functions como proxy autenticado entre o GPT Action e os Web Services do Moodle.

O responsável pelo tratamento dos dados é o operador do projeto, identificável pelo domínio de hospedagem desta página. Para exercer direitos ou solicitar informações sobre privacidade, utilize o canal de contato indicado na Seção 17.

2. Abrangência desta política

Esta Política de Privacidade aplica-se a todos os componentes do projeto:

  • Página de cadastro, login e painel (frontend React + Vite).
  • Edge Function platform-api: autenticação, gerenciamento de chaves de API, ativação de serviços e schemas OpenAPI.
  • Edge Function moodle-proxy: validação da chave e execução de consultas somente leitura no Moodle.
  • Schema OpenAPI disponibilizado para importação como GPT Action no ChatGPT.

Esta política não substitui os termos de privacidade do Moodle, do ChatGPT/OpenAI, do Supabase ou de outros provedores de infraestrutura utilizados.

3. Dados pessoais que tratamos

3.1 Dados de cadastro e autenticação na plataforma

  • Nome completo e endereço de e-mail.
  • Hash HMAC-SHA-256 da chave de API e preview parcial para exibição.
  • Status da chave (ativa/inativa), data de criação e data do último uso.
  • Metadados técnicos da sessão gerenciados pelo Supabase Auth (JWT, refresh token, timestamps).

3.2 Dados técnicos de uso da API

  • Rotas consultadas e parâmetros enviados (ex.: IDs de curso, IDs de usuário, e-mails, termos de busca).
  • Data, hora e resultado das requisições.
  • Mensagens de erro e metadados de diagnóstico.
  • Dados técnicos registrados pelos provedores de infraestrutura (endereço IP, user agent, logs de acesso).

3.3 Dados acadêmicos retornados pelo Moodle

O proxy pode retornar, sob demanda, os dados disponíveis no Moodle conforme a sessão ativa do usuário. Dependendo das rotas consultadas e das permissões configuradas, esses dados podem incluir:

  • Dados de cursos, turmas, seções, conteúdos e atividades.
  • Dados de participantes (ID Moodle, username, nome, e-mail, papel no curso, datas de acesso).
  • Notas, itens de nota, progresso e status de conclusão.
  • Entregas, submissões, prazos e relatórios de pendências.
  • Dados retornados por relatórios configuráveis do Moodle, quando autorizados.

O Moodle Consultas é projetado como proxy somente leitura. Não cria, altera nem exclui usuários, cursos, matrículas, notas, mensagens ou atividades no Moodle.

4. Finalidades do tratamento

  • Cadastrar usuários autorizados e gerar chaves de API para acesso à plataforma.
  • Autenticar requisições e permitir consultas acadêmicas autorizadas ao Moodle.
  • Apoiar análises institucionais sobre cursos, alunos, notas, conclusão, acessos e pendências.
  • Proteger o ambiente contra uso indevido, acessos não autorizados e exposição de segredos.
  • Diagnosticar erros, monitorar disponibilidade e manter a segurança da API.
  • Cumprir obrigações legais, regulatórias, contratuais ou institucionais aplicáveis.

5. Bases legais para o tratamento

O tratamento de dados pessoais nesta plataforma pode estar fundamentado nas seguintes bases legais, conforme o contexto institucional de cada usuário:

  • Execução de contrato ou procedimentos relacionados ao vínculo educacional ou institucional.
  • Cumprimento de obrigação legal ou regulatória.
  • Legítimo interesse do responsável pelo tratamento, respeitados os direitos e liberdades dos titulares.
  • Consentimento, quando a instituição optar por utilizá-lo em fluxos específicos.

6. Compartilhamento de dados

Os dados podem ser acessados ou compartilhados nos seguintes contextos:

  • Moodle institucional: fonte primária dos dados acadêmicos consultados.
  • Supabase: hospedagem das Edge Functions, armazenamento de chaves de API e autenticação.
  • OpenAI / ChatGPT: quando o usuário configura e utiliza um GPT Action para consultar a API. O tratamento de dados pela OpenAI é regido pelos termos de privacidade da OpenAI.
  • Equipes internas autorizadas: tecnologia, gestão acadêmica, coordenação, segurança da informação, auditoria ou suporte.
  • Autoridades públicas: quando houver obrigação legal ou ordem válida.

Segredos operacionais — como tokens Moodle cifrados, service role keys e chaves de hash — não são expostos no schema OpenAPI, no frontend, em respostas da API nem em logs públicos.

7. Retenção e exclusão de dados

Os dados de cadastro e chave de API são armazenados no Supabase enquanto forem necessários para autenticar o usuário, manter auditoria operacional e cumprir obrigações aplicáveis. A chave completa não é persistida — apenas o hash e o preview.

Os dados acadêmicos consultados no Moodle são retornados sob demanda pelo proxy; a plataforma não mantém cópia permanente desses dados, salvo registros técnicos ou logs de infraestrutura necessários para segurança e diagnóstico.

Quando uma chave de API deixar de ser necessária, poderá ser desativada ou excluída conforme os procedimentos internos e as obrigações de retenção aplicáveis.

8. Segurança da informação

Adotamos medidas técnicas e organizacionais proporcionais ao risco, incluindo:

  • Autenticação por chave de API enviada como Bearer token (formato gah_live_*).
  • Armazenamento das chaves apenas como HMAC-SHA-256 — a chave completa é exibida somente uma vez.
  • Tokens Moodle armazenados cifrados como secrets de backend no Supabase.
  • Limitação das funções Moodle permitidas a operações somente leitura.
  • Controle de acesso via Row Level Security (RLS) no banco de dados.
  • Recomendação de não inserir senhas, tokens ou segredos em conversas, prompts ou parâmetros visíveis da API.

9. Uso de GPT Actions e Inteligência Artificial

O Moodle Consultas permite que um GPT Action consulte dados do Moodle por meio de endpoints autenticados. Ao usar esse recurso:

  • O usuário deve consultar apenas dados necessários para sua finalidade autorizada.
  • Dados pessoais não devem ser solicitados ou exibidos além do necessário.
  • O GPT não deve receber senhas, tokens Moodle, service role keys ou outros segredos.
  • Respostas geradas por IA devem ser conferidas antes de serem usadas em decisões acadêmicas ou institucionais relevantes.

O uso do ChatGPT e dos GPT Actions é regido pelos termos e pela política de privacidade da OpenAI, que realiza tratamentos próprios conforme suas regras.

10. Menores de idade e dados educacionais

Como o Moodle pode conter dados de estudantes, inclusive eventualmente menores de idade, as consultas devem respeitar as permissões institucionais e a necessidade de cada finalidade educacional. O acesso a dados de alunos é restrito a pessoas autorizadas e deve ser utilizado exclusivamente para fins legítimos relacionados a acompanhamento, gestão, suporte, avaliação ou obrigações institucionais.

11. Transferência internacional de dados

Dependendo da localização dos provedores de infraestrutura utilizados (Supabase, OpenAI/ChatGPT, GitHub), dados pessoais podem ser tratados ou armazenados fora do Brasil. Nesses casos, o responsável pelo tratamento deve verificar a existência de mecanismos jurídicos adequados e medidas de segurança compatíveis com a LGPD.

12. Cookies e rastreamento

A plataforma não utiliza cookies próprios de rastreamento ou publicidade. Provedores de hospedagem, navegadores ou ferramentas de infraestrutura podem registrar dados técnicos de acesso para segurança e operação. Caso sejam adicionadas ferramentas de analytics ou monitoramento, esta política será atualizada antes da ativação desses recursos.

13. Direitos dos titulares

Nos termos da Lei Geral de Proteção de Dados Pessoais (LGPD — Lei n.º 13.709/2018), os titulares podem solicitar:

  • Confirmação da existência de tratamento.
  • Acesso aos dados pessoais.
  • Correção de dados incompletos, inexatos ou desatualizados.
  • Anonimização, bloqueio ou eliminação de dados desnecessários ou tratados em desconformidade.
  • Portabilidade dos dados, quando regulamentada e aplicável.
  • Informações sobre compartilhamento de dados com terceiros.
  • Revogação do consentimento, quando esta for a base legal utilizada.
  • Revisão de decisões tomadas unicamente com base em tratamento automatizado, quando houver.

Solicitações devem ser encaminhadas ao canal de contato indicado na Seção 17. O atendimento poderá exigir validação de identidade.

14. Responsabilidades do usuário autorizado

O usuário que recebe uma chave de API assume as seguintes responsabilidades:

  • Guardar a chave em local seguro e não compartilhá-la com terceiros não autorizados.
  • Usar a API exclusivamente para finalidades institucionais autorizadas.
  • Consultar somente os dados necessários para cada finalidade.
  • Não exportar, copiar ou divulgar dados pessoais fora dos ambientes autorizados.
  • Comunicar imediatamente qualquer suspeita de vazamento, perda de chave ou acesso indevido.

15. Alterações desta política

A data de última atualização no topo desta página reflete a versão mais recente. Recomendamos a revisão periódica deste documento.
